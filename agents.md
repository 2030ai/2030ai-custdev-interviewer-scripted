# CustDev Interviewer

Автоматизированная система для валидации стартап-идей через генерацию custdev-интервью и анализ результатов.

## Описание

Этот проект помогает быстро проверить стартап-идею, создавая симулированные custdev-интервью с целевой аудиторией. На выходе вы получаете:
- Аналитику по результатам 10 интервью
- Рекомендации по запуску стартапа
- 3 готовых сценария продающих роликов

## Как это работает

Система использует последовательность промптов из папки `prompts/`, применяя их к вашей стартап-идее. Все промежуточные результаты сохраняются в отдельной папке для каждой идеи.

## Пошаговая инструкция

### Пример: Идея "Школа AI курсов"

**Шаг 0: Подготовка**
1. Создайте папку для вашей идеи: `outputs/школа-ai-курсов/`
2. Все сгенерированные файлы будут сохраняться в этой папке с названиями по номеру промпта

---

### Шаг 1: Структурирование стартапа

**Промпт:** `prompts/1 startup mix.md`

**Входные данные:**
- Ваша стартап-идея: "Школа AI курсов"

**Действие:**
1. Используйте промпт из `1 startup mix.md`
2. Примените его к вашей идее
3. Получите структурированное описание: Аудитория, Проблема, Решение

**Сохранить в:** `outputs/школа-ai-курсов/1-startup-mix.md`

**Пример результата:**
```
Аудитория: Middle разработчики, 2-5 лет опыта, 150-250k/год, хотят внедрять AI в свои проекты
Проблема: Нет систематизированных знаний для практического применения AI в production
Решение: 8-недельный онлайн-курс с практическими проектами, на выходе - готовое AI-приложение в портфолио
```

2. Что вас останавливает от более активного использования AI?
3. Сколько времени готовы инвестировать в обучение?

### Шаг 4: Создание персон (10 штук)

**Промпт:** `prompts/4 person.md`

**Входные данные:**
- Описание аудитории из шага 1

**Действие:**
1. Используйте промпт из `4 person.md`
2. Сгенерируйте 10 различных персон на основе описания аудитории
3. Каждая персона должна иметь уникальные характеристики

**Сохранить в:**
- `outputs/школа-ai-курсов/4-person-01.md`
- `outputs/школа-ai-курсов/4-person-02.md`
- ...
- `outputs/школа-ai-курсов/4-person-10.md`

**Пример результата (персона 1):**
```
Имя: Алексей Петров
Возраст: 28 лет
Должность: Middle Python Developer в Яндексе
Опыт: 3.5 года в разработке
Зарплата: 220,000₽/месяц
Город: Москва

Контекст:
Работает в команде рекомендательных систем, видит как коллеги используют ML, но сам владеет только базовыми навыками. Хочет расти в сторону AI/ML, но не понимает с чего начать в своем контексте.

Боли:
- Теряет возможности карьерного роста из-за отсутствия AI-компетенций
- Не может эффективно коммуницировать с ML-командой
- Пробовал учиться сам по YouTube, но знания разрозненные
```

### Шаг 5: Генерация интервью (10 штук)

**Промпт:** `prompts/5 script.md`

**Входные данные:**
- Персона из шага 4
- Вопросы из шага 3

**Действие:**
1. Используйте промпт из `5 script.md`
2. Для каждой из 10 персон сгенерируйте дословный скрипт интервью
3. Ответы должны соответствовать характеру и контексту персоны

**Сохранить в:**
- `outputs/школа-ai-курсов/5-script-01.md`
- `outputs/школа-ai-курсов/5-script-02.md`
- ...
- `outputs/школа-ai-курсов/5-script-10.md`

**Пример результата (интервью 1 с Алексеем):**
```
Интервьюер: Как сейчас решаете задачи, связанные с AI в ваших проектах?

Алексей: Если честно, пока что обращаюсь к коллегам из ML-отдела. У меня есть базовое понимание, могу подключить готовые модели через API, но самостоятельно разработать или даже настроить модель под задачу - нет. Это неудобно, потому что создает зависимость и замедляет работу.

Интервьюер: Что вас останавливает от более активного использования AI?

Алексей: Главное - не знаю с чего начать конкретно для моих задач. В интернете миллион курсов, но они либо слишком теоретические с математикой, либо поверхностные "как запустить ChatGPT". Мне нужно что-то прикладное для backend-разработки.

...
```

### Шаг 6: Анализ интервью

**Промпт:** `prompts/6 analyse.md`

**Входные данные:**
- Все 10 скриптов интервью из шага 5
- Список вопросов из шага 3

**Действие:**
1. Используйте промпт из `6 analyse.md`
2. Соберите все 10 интервью в один запрос
3. Получите аналитику: таблицу ответов, причины ЗА/ПРОТИВ, итоговый совет

**Сохранить в:** `outputs/школа-ai-курсов/6-analyse.md`

**Пример результата:**
```
## Анализ результатов custdev-интервью

### Таблица частых ответов по вопросам

| Вопрос | Наиболее частые ответы | Количество |
|--------|------------------------|------------|
| Как сейчас решаете задачи с AI? | Обращаюсь к коллегам/использую готовые API | 7/10 |
| Что останавливает? | Нет структурированного обучения для практики | 8/10 |
| Сколько времени готовы инвестировать? | 2-3 месяца по 5-10 часов в неделю | 6/10 |
...

### Три причины ЗАПУСКАТЬ стартап:

1. **Явный болевой запрос**: 8 из 10 респондентов активно ищут способ системно изучить AI для практического применения
2. **Готовность платить**: 9 из 10 готовы инвестировать 30,000-80,000₽ в обучение
3. **Недостаток конкурентных решений**: Существующие курсы либо слишком теоретические, либо поверхностные

### Три причины НЕ ЗАПУСКАТЬ стартап:

1. **Высокие ожидания по времени**: Аудитория хочет результат за 2-3 месяца, но реальное освоение AI требует больше времени
2. **Разнородность потребностей**: Разработчики работают в разных стеках (Python, JS, Go) и хотят разного
3. **Скептицизм к онлайн-форматам**: 4 из 10 уже проходили онлайн-курсы и были разочарованы

### Итоговый совет:

**ЗАПУСКАТЬ, но с модификацией**.

Есть четкий спрос от платежеспособной аудитории. Но критически важно:
- Фокус на конкретном стеке (начать с Python для backend/data)
- Упор на практические проекты, а не теорию
- Сильное комьюнити и менторство для снижения скептицизма
- Реалистичные сроки (12 недель вместо 8)

Рекомендуется запустить MVP-курс на 20-30 студентов для валидации.
```

### Шаг 7: Генерация сценариев роликов

**Промпт:** `prompts/7 reels.md`

**Входные данные:**
- Аудитория/Проблема/Решение из шага 1
- Инсайты из анализа (шаг 6)

**Действие:**
1. Используйте промпт из `7 reels.md`
2. Сгенерируйте 3 разных сценария продающих роликов
3. Каждый сценарий должен использовать инсайты из интервью

**Сохранить в:** `outputs/школа-ai-курсов/7-reels.md`

**Пример результата:**
```
## Сценарий 1: "Проблема-Решение" (30 секунд)

[Кадр 1 - 3 сек]
Текст на экране: "Коллеги уже делают AI-фичи..."
Закадровый: Middle разработчик, а до сих пор просишь ML-команду помочь?

[Кадр 2 - 5 сек]
Визуал: Фрустрированный разработчик смотрит на код с AI
Текст: "YouTube не помог. Курсы слишком теоретические"

[Кадр 3 - 7 сек]
Визуал: Превью интерфейса курса
Закадровый: 8 недель. Реальные AI-проекты. Zero теории без практики.

...

## Сценарий 2: "История успеха" (45 секунд)

[Кадр 1]
Персонаж: Алексей, Python dev
"3 месяца назад я даже не знал как работает нейросеть"

...

## Сценарий 3: "Цифры и факты" (20 секунд)

[Кадр 1]
Текст на экране: "80% разработчиков хотят работать с AI"

[Кадр 2]
Текст: "Но только 15% умеют это делать на практике"

...
```

## Советы по использованию

- **Будьте конкретны** в формулировке идеи - чем конкретнее, тем лучше результат
- **Разнообразьте персон** на шаге 4 - делайте их с разным опытом, контекстом и болями
- **Сохраняйте промежуточные результаты** - они помогут понять инсайты и улучшить следующие итерации
- **Используйте инсайты из анализа** при создании сценариев роликов
